# Контрольные вопросы retrieval

`retrieval_cases.json` содержит контрольные вопросы для локальной оценки семантического поиска LawyerChat без участия LLM.

Каждый кейс задает:

- уникальный `id`;
- название закона;
- ожидаемые идентификатор и название документа;
- один или несколько ожидаемых номеров статей;
- вопрос пользователя;
- комментарий для ручной проверки.

Чтобы добавить вопрос, внесите новый объект в JSON-массив. Номера статей указывайте строками в `expected_article_numbers`, например `["18"]` или `["18", "21", "22"]`.

Запуск из папки `backend`:

```powershell
python -m scripts.evaluate_retrieval `
  --cases "data/evaluation/retrieval_cases.json" `
  --top-k 20 `
  --output-dir "reports"
```

По умолчанию используются те же пути и `top_k=20`. Если указать меньшую глубину, метрики выше `top_k` будут отмечены как не рассчитанные. CSV, JSON, Markdown и DOCX-отчеты сохраняются в `backend/reports/`.
