# Юридические документы

Сюда складываются `.jsonl` файлы, полученные после конвертации исходных HTML/HTM документов из `backend/data/raw_html/`.

Форматы:

- `.jsonl` — основной структурированный режим, каждая строка является отдельным готовым юридическим фрагментом.
- `.txt` — legacy/fallback режим, файл разбивается на чанки через `split_legal_text`.

TXT больше не рекомендуется для нормативно-правового корпуса, потому что теряет структуру разделов, глав и статей.

Для конвертации HTML в JSONL из папки `backend`:

```bash
python -m scripts.convert_html_to_jsonl --input-dir "data/raw_html" --output-dir "data/legal_docs"
```

После добавления или обновления файлов запустите индексацию:

```bash
python -m scripts.ingest_documents
```

Или используйте endpoint:

```text
POST /api/documents/reindex
```

Не добавляйте сюда автоматически скачанные документы. База текстов для MVP наполняется вручную.
